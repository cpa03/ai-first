## 2026-02-19 - [Keyboard Shortcuts Modal Enhancements]
**Learning:** Platform-specific UI elements (like shortcut keys) can cause hydration mismatches in Next.js if they rely on `navigator` during the initial render. Using a `useEffect` to set platform state after mounting is the standard fix, but it's important to ensure that constants used in calculations aren't erroneously added to hook dependency arrays, as this triggers ESLint warnings. Adding hover states to list items in modals significantly improves the "focus" feel for users navigating with a mouse.
**Action:** Always use `useEffect` for platform detection in client components. When adding hover effects to items that span the full width of a container, use negative horizontal margins combined with horizontal padding to create a "bleed" effect that feels more polished.
