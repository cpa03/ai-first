## 2026-07-21 - [Sanitization Bounded Map Cache]
**Learning:** High-frequency input validation and dynamic data recursive sanitization (such as `sanitizeObject` and `sanitizeHtml`) trigger massive CPU overhead due to sequential regular expression execution. By implementing a Map-based memoization cache with an O(1) eviction policy (using the Map's insertion order), we can bypass regex execution entirely for identical strings, achieving up to ~27x speedup for unsafe strings on subsequent runs, without risking memory leaks.
**Action:** Always consider bounded O(1) Map caching for deterministic, stateless functions like HTML sanitization, string parsing, or cryptography fallbacks in hot code paths. Use Map insertion order (`Map.keys().next().value`) to easily evict the oldest entries without the overhead of tracking hit frequency or sorting.
