# Documentation Archive

This directory contains archived documentation that is no longer actively maintained but preserved for historical reference.

## Purpose

- **Historical Reference**: Documents that document past decisions, deprecated features, or superseded specifications
- **Audit Trail**: Preserved for compliance and historical analysis
- **Knowledge Base**: Valuable context that may inform future decisions

## Archive Guidelines

### What to Archive

- Superseded design documents
- Deprecated feature documentation
- Old version specifications
- Completed project phase documentation
- Historical meeting notes and decisions

### What NOT to Archive

- Active documentation (belongs in parent docs/ directory)
- Sensitive information (should be removed entirely)
- Temporary files (use .gitignore instead)

## File Naming Convention

Archive files should follow this naming pattern:

```
YYYY-MM-DD_original-filename_archived.md
```

Example: `2025-01-15_old-api-design_archived.md`
